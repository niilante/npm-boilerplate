Getting started

Clone this repository and use it as a starting point.

git clone git@github.com:pixelass/npm-boilerplate.git my-npm-module
cd my-npm-module
## use yarn
yarn
## or npm
## npm i

Reset git

cd my-npm-module
rm -rf .git
git init

Run package config

Use the package.config.json file to enter your own package data. Then simple run the init script to update the original package.json of this project

node scripts/npm-init

after that you can remove the config files

rm -rf package.config.json scripts/npm-init.js

Developing

To start a dev server and start developing try the following commands

  • start: starts the dev server and builds the required files
  • test: runs test and lints files
  • run dev: starts the dev server and watches the required files
  • run babel: generates lib from source
  • run build: builds all files from source
  • run watch: builds and watches all files from source
  • run lint: lints javascript files
  • run release: release new version using "standard-version"

Examples

Starts a simple http-server

npm start

Starts a simple http-server and watches files

npm run dev

What's included?

Transforms

  • Babel
    • stage 0
    • es2015
  • Browserify

You can change the rules inside the package.json file.

  • babel: {<SETTINGS>}
  • browserify: {<SETTINGS>}

defaults

{
  "babel": {
    "presets": [
      "es2015",
      "stage-0"
    ]
  },
  "browserify": {
    "transform": [
      "babelify"
    ]
  }
}

Coding style

  • CSS: stylelint
  • JS: xo

You can change the rules inside the package.json file.

  • xo: {<SETTINGS>}
  • stylelint: {<SETTINGS>}

defaults

{
  "xo": {
    "space": true,
    "semicolon": false
  },
  "stylelint": {
    "extends": "stylelint-config-standard",
    "rules": {
      "indentation": 2,
      "number-leading-zero": null
    }
  }
}

Code quality

  • bithound

You can change the rules inside the .bithound file.

defaults

{
  "ignore": [
    "**/node_modules/**",
    "**/**.min.**",
    "**/dist/**",
    "**/docs/**",
    "/index.js"
  ],
  "test": [
    "**/spec/**"
  ],
  "critics": {
    "lint": {
      "engine": "none"
    },
    "wc": {
      "limit": 5000
    }
  }
}

Testing

  • Ava
  • Coveralls
  • tap-nyan (for some fun)

You can change the rules inside the package.json file.

  • ava: {<SETTINGS>}

defaults

{
  "ava": {
    "files": [
      "src/**/spec/*.js"
    ],
    "source": [
      "src/**/*.js"
    ],
    "presets": [
      "@ava/stage-4",
      "@ava/transform-test-files"
    ],
    "failFast": true,
    "tap": true,
    "require": [
      "babel-register"
    ],
    "babel": "inherit"
  }
}

Documentation

  • jsdoc
  • hopsdoc

You can change the rules inside the jsdoc.json file.

defaults

{
  "tags": {
    "allowUnknownTags": true,
    "dictionaries": ["jsdoc"]
  },
  "plugins": [
    "plugins/markdown"
  ],
  "templates": {
    "cleverLinks": true,
    "monospaceLinks": true
  },
  "source": {
    "include": ["src", "README.md"],
    "includePattern": "\\.js$",
    "excludePattern": "(node_modules/|docs|coverage|lib)"
  },
  "opts": {
    "encoding": "utf8",
    "destination": "./docs/api",
    "private": false,
    "recurse": true,
    "template": "node_modules/hopsdoc"
  }
}

Hopsdoc is a fully responsive theme that allows custom themes. You can simply remove it and load a different theme.

yarn remove hopsdoc

CSS (optional)

  • css-modulesify (disabled)
  • cssnext (disabled)

To enable css-modules simply install the dependency and follow the comments inside scripts/run-build.js

yarn add css-modulesify -D

To enable cssnext simply install the dependency and follow the comments inside scripts/run-build.js

yarn add postcss-cssnext -D
